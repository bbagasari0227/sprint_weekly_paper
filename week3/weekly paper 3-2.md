# Tableau 인터랙티브 대시보드 활용 사례  
## 병원 관리자를 위한 병상 및 환자 관리 대시보드 설계

정적인 차트는 전체적인 현황을 보여주는 데에는 효과적이지만 
사용자가 원하는 조건에 따라 데이터를 직접 탐색하기에는 한계가 있다.  

특히 의료 데이터는 시간, 환자 상태, 병실 상황에 따라 지속적으로 변화하기 때문에  
인터랙티브 기능을 통한 동적 분석이 중요하다.

---

## 1. 활용 상황

병원 관리자가 병상 운영과 환자 수용 현황을 관리하는 상황을 가정한다.  

관리자는 다음과 같은 정보를 빠르게 파악해야 한다.

- 병실 유형별(1인실, 3인실, 6인실) 가격
- 병실별 공석 현황
- 진료과별 병실 분포
- 중환자실 및 일반 병실 수용 가능 여부
- 시간에 따른 병상 사용 변화

---

## 2. 적용할 인터랙티브 기능

### (1) 병실 유형 필터

#### 기능
- 1인실 / 3인실 / 6인실 선택

#### 이유
필터가 없을 경우 모든 병실 정보가 섞여 평균값 중심으로 보이게 된다.  
이 경우 병실 유형별 가격 차이나 공석 현황을 정확히 파악하기 어렵다.  

따라서 병실 유형을 구분하여 분석할 수 있도록 한다.

---

### (2) 진료과 필터

#### 기능
- 내과, 외과 등 진료과 선택

#### 이유
진료과별로 병실 수와 환자 수용 상황이 다르기 때문에  
특정 과에 집중하여 병상 운영 현황을 파악할 수 있도록 한다.

---

### (3) 시간(날짜/시간대) 필터

#### 기능
- 날짜, 요일, 시간대 선택

#### 이유
병실 이용률과 환자 입·퇴원은 시간에 따라 크게 달라진다.  

예를 들어,
- 특정 시간대에 병실 부족 현상이 발생할 수 있고  
- 특정 요일에 입원 환자가 집중될 수 있다  

따라서 시간 조건을 기준으로 데이터를 분석할 수 있도록 한다.

---

### (4) 클릭 액션 (Drill-down)

#### 기능
- 특정 병실 유형 또는 진료과를 클릭하면  
  → 해당 조건의 상세 정보 표시

예:
- 병실 유형 클릭 → 공석 수, 가격 정보 확인  
- 진료과 클릭 → 병실 수, 중환자실 여부 확인  
- 특정 시간대 선택 → 해당 시간 기준 병상 상태 확인  

#### 이유
전체 현황을 보는 것에서 나아가  
특정 조건에 대한 상세 분석을 빠르게 수행할 수 있도록 하기 위함이다.

---

## 3. 인터랙티브 기능의 효과

이러한 기능을 통해 병원 관리자는 다음과 같은 의사결정을 할 수 있다.

- 병실 공석을 기반으로 입원 환자 배치 최적화  
- 병실 유형별 가격과 수요를 고려한 운영 전략 수립  
- 특정 시간대의 병상 부족 문제 사전 대응  
- 응급 환자 수용 가능 여부를 신속하게 판단  

즉, 데이터를 단순히 확인하는 것이 아니라  
실제 운영 의사결정에 직접 활용할 수 있다.

---

## 4. 결론

의료 데이터는 시간에 따라 지속적으로 변화하며  
상황에 맞는 빠른 판단이 중요한 영역이다.  

Tableau의 필터와 액션 기능을 활용하면  
사용자가 원하는 조건(병실 유형, 진료과, 시간 등)에 따라  
데이터를 직접 탐색할 수 있으며  

이를 통해 보다 효율적이고 현실적인 병원 운영이 가능해진다.
