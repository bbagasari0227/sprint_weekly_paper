# 결측값 처리 전략: 단순 제거가 위험한 이유와 의사결정 기준

## 1. 결측값을 무조건 제거하는 것이 위험한 이유

결측값을 단순히 제거하는 방식은 구현이 쉽지만 아래와 같은 문제를 유발할 수 있다.

### (1) 표본 수 감소
결측값을 포함한 행을 제거하면 전체 데이터 수가 크게 줄어들 수 있다.  
이는 특히 데이터셋이 작은 경우 분석 자체가 어려워질 수 있다.

### (2) 데이터 대표성 훼손
결측값이 특정 집단에서 집중적으로 발생한 경우 해당 집단의 데이터가 불균형적으로 제거될 수 있다.  
→ 결과적으로 **분석 결과가 전체 모집단을 대표하지 못하게 된다**

### (3) 데이터 편향 발생
결측이 무작위가 아니라면 데이터를 제거하는 순간 특정 패턴이 사라지게 된다.  
→ 모델이 잘못된 방향으로 학습될 가능성이 있다.

### (4) 집단 특성 왜곡
특정 변수(예: 연봉, 나이 등)의 결측값을 가진 데이터가 제거되면  
→ 데이터의 분포 자체가 바뀌고 원래의 집단 특성이 왜곡될 수 있다.

---

## 2. 결측값 처리 방법을 결정하는 기준

나는 결측값을 처리할 때 다음 기준을 기반으로 의사결정을 한다.

### (1) 결측 비율 확인
- 결측값이 매우 소수(예: 5% 미만)인 경우 → 제거 고려한다.
- 결측값 비율이 높은 경우 → 대체 방법 고려한다.

### (2) 결측 메커니즘 파악
- **MCAR (완전 무작위 결측)**: 제거해도 비교적 안전하다.
- **MAR / MNAR**: 제거 시 편향 발생 가능 → 신중한 접근 필요하다.

### (3) 변수의 중요도 고려
- 모델에서 중요한 변수라면 → 제거보다 대체를 우선 고려한다.
- 영향이 거의 없는 변수라면 → 제거 가능하다.

### (4) 도메인 지식 활용
- 데이터의 의미를 이해하고 판단한다.
- 예: 특정 값이 비어있는 것이 실제로 의미를 가지는 경우도 있다.

---

## 3. 결측값 대체 방법

데이터 타입과 특성에 따라 다음과 같이 처리한다.

### (1) 수치형 데이터
- **나이** → 평균값으로 대체한다.
- **연봉** → 이상치 영향을 줄이기 위해 중앙값 사용한다.

### (2) 범주형 데이터
- **성별** → 최빈값으로 대체한다.

---

## 4. 결론

결측값 처리는 단순히 제거하는 문제가 아니라  
**데이터의 대표성과 분석 결과의 신뢰성을 결정하는 중요한 과정**이다.

따라서 결측 비율, 데이터 특성, 도메인 지식을 종합적으로 고려하여 상황에 맞는 처리 전략을 선택하는 것이 중요하다.
